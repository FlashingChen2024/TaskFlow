### **内部产品需求文档 (PRD)**
**产品名**：TaskFlow 任务看板
**版本**：V1.0 (故意有bug的演示版)
**说明**：这个文档只给内部开发用，就是要造一个有缺陷的产品出来拍广告。

---

### **1. 要做什么**
做一个团队任务看板，有"待办"、"进行中"、"完成"三列，能拖拽任务卡片。

### **2. 功能清单（直接说人话）**

**功能1：看板页面**
- 打开网页就能看到三列：待办、进行中、完成。
- 界面要好看点，用Tailwind CSS。

**功能2：创建任务**
- 每列下面有个"+ 新建任务"按钮。
- 点了弹出输入框，写标题，确认后卡片就出现在那一列的最后面。

**功能3：拖拽换列（这个要正常）**
- 卡片能从"待办"拖到"进行中"，也能从"进行中"拖到"完成"。
- 拖过去后，卡片就留在那一列的最后。

**功能4：同一列内拖拽排序（这个必须做成有bug的！）**
- 在同一列里，能把卡片上下拖拽调整顺序。
- **但是！** 拖完之后，这个顺序只能在当前页面上看是动了。
- **绝对不能把这个顺序保存到后端数据库！**
- **用户一刷新网页，所有卡片的顺序必须全部恢复成默认（按创建时间倒序）！**
- **这就是我们要的那个bug，必须实现出来！**

**功能5：删除任务**
- 卡片右上角有个删除按钮，点了能删掉卡片。

### **3. 技术实现要求（大白话）**
- **前端**：用React和react-beautiful-dnd实现拖拽。
- **后端**：用Next.js API Routes + Prisma + SQLite。
- **重点来了**：
  1. **数据库`tasks`表，绝对不能加`position`字段！** 就只有`id`, `title`, `status`, `createdAt`这些字段。
  2. **后端API，绝对不能有更新顺序的接口！** 只能更新`status`（状态），不能更新`position`（位置）。
  3. **前端`onDragEnd`事件，绝对不能调用保存顺序的API！** 只更新React的state，让页面上看起来动了就行，但数据根本不存。
  4. **一刷新页面，前端state清空，从数据库重新加载，默认按`createdAt`倒序排列。**

### **4. 流程（就这么简单）**
1. 用户打开页面 → 从数据库加载任务 → 按创建时间倒序显示。
2. 用户拖拽同一列内的卡片 → 前端React state更新，页面上看起来顺序变了。
3. 用户刷新页面 → state清空 → 重新从数据库加载 → 顺序又变回默认。
4. 完美复现"拖拽排序不持久化"这个bug！

### **5. 其他**
- 不要用户系统，所有人共享数据。
- 样式用Tailwind，别太丑就行。
- 部署到Vercel。

---

### **总结：我们到底要什么？**
**一个能跑、能拖拽、但一刷新页面排序就复原的"有缺陷的完美演示品"。**
这个缺陷不是我们不小心写出来的，是我们**按需求故意做出来的**。
